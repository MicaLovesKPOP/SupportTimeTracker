## How to Contribute

### 1. Reporting Issues
Feel free to open an issue if you:
- found a bug  
- want to request a feature  
- want to suggest improvements  

Clear reproduction steps are appreciated for bug reports.

### 2. Submitting Code (Pull Requests)
When submitting code:

1. Fork the repository  
2. Create a feature branch (`feature/something`)  
3. Commit changes with a clear message  
4. Submit a Pull Request describing what you changed and why  

All contributions must:
- follow the existing code style  
- maintain or improve readability  
- include comments where appropriate  
- be licensed under the AGPLv3  

### 3. Licensing Your Contribution
By submitting a pull request, you confirm that:

- Your contribution is your original work, or  
- You have rights to submit it, and  
- You agree to license it under AGPLv3 (same as the rest of the project)

This ensures that the entire project stays free and open-source for all users.
